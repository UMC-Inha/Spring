## 접근제어자

---

### public

접근제어자 중 1순위로 권한이 넓으며, 모든 클래스에서의 접근을 가능하도록 함.

```jsx
public class A{
	public int a;
	public void AA(){
		...
	}
	...
}

//a, AA()는 모든 클래스에서 접근 가능
```

### protected

접근제어자 중 2순위로 권한이 넓으며, 자식 클래스와 같은 패키지 내에 있는 클래스에서의 접근을 가능하도록 함.

```jsx
package one;

public class A{
	protected int a;
	protected void AA(){
		...
	}
	...
}

package two;
public class B extends A{
	//a, AA() 접근 가능
}

package one;
public class C{
	//a, AA() 접근 가능
}

```

### default

접근제어자 중 3순위로 권한이 넓으며, 같은 패키지 내에 있는 클래스에서의 접근을 가능하도록 함. 접근제어자를 명시하지 않으면 default로 지정됨.

```jsx
package one;
public class A{
	int a;
	void AA();
	...
}

package one;
public class B{
	//a, AA 접근 가능
}
```

### private

접근제어자 중 권한이 가장 좁으며, 해당 클래스에서만 접근이 가능함.

```jsx
package one;
public class A{
	private int a;
	private void AA(){
		...
	}
	...
}

//a, AA는 A클래스에서만 접근 가능
```

## 클래스와 인터페이스

---

### 클래스

- 객체의 속성과 동작을 정의하는 틀
- 필드, 메서드를 바탕으로 객체를 정의
- 클래스를 이용해 인스턴스화하여 객체를 생성

### 인터페이스

- 순수 추상 클래스를 더 편리하게 사용할 수 있는 기능
- 클래스가 구현해야하는 메서드 시그니처를 모아놓은 것.
- 클래스에서 인터페이스의 메서드를 구현해야 함

## 상속

---

- 클래스가 다른 클래스의 필드 및 메서드를 재사용 할 수 있게 하는 기능
- 부모(슈퍼) 클래스: 상속을 통해 자신의 필드 및 메서드를 제공해주는 클래스
- 자식(서브) 클래스: 부모 클래스를 통해 필드와 메서드를 상속받는 클래스
- 자식 클래스는 상속받은 기능을 그대로 사용할 수도, 변경 및 확장할 수도 있음
- 자바는 다중상속을 지원하지 않음

## 면접 질문

---

- **자바가 다중상속을 지원하지 않는 이유**  
  → 여러 개의 부모를 상속받았을 때, 동일한 메서드가 있으면 어떤 메서드를 상속받아야 할지 모호해지는 상황을 막기 위함.

- **인터페이스에서는 다중상속을 지원하는 이유**  
  → 다중상속 받은 인터페이스에 동일한 메서드가 존재하더라도, 클래스에서 반드시 메서드를 구현해야하므로 클래스의 다중상속과는 다르게 모호한 상황을 피할 수 있음.  
 →인터페이스의 다중상속을 지원함으로써 다형성을 높이고 유연한 설계가 가능하도록 함.
