# 🎯핵심 키워드 정리
## IP
  - OSI 7계층에서 Network layer에 위치하는 프로토콜을 의미한다.
  - 패킷(packet)이라는 통신 단위로 데이터 전달하며, 지정한 IP 주소(IP Address)에 데이터 전달한다.

#### IP 패킷이란?
  - IP 패킷은 헤더와 페이로드 두 부분으로 구성되어있다.
  - 헤더에는 출발지 IP 주소, 목적지 IP 주소와 같은 패킷 정보가 포함되고
  - 페이로드에는 실제 전송 데이터가 포함된다.

#### IP 주소란? 
  - 인터넷에 연결된 각 기기들을 구별하는 식별자이다.
  - IP 주소는 숫자 형태이므로, 사람이 이해할 수 있도록 도메인 이름(Domain Name)을 지정한다.

#### IP 주소의 버전
> 예시) 
> <br>IPv4 : 192.168.0.1
> <br>IPv6 : 2001:0db8:85a3:0000:0000:8a2e:0370:7334
- IP 주소는 IPv4, IPv6 두 개의 버전이 존재한다.
  - IPv4는 32 bit의 주소 체계를 가지고 있어 약 42억 개의 고유 주소를 발급할 수 있다.
  - IPv6는 128 bit의 주소 체계를 가지고 있어 IPv4에 비해 훨씬 많은 주소를 확보할 수 있다.

#### IP 주소의 구성
> 참고 동영상 : https://www.youtube.com/watch?v=b7Wk-6w5vgg&t=460s
  - IP 주소는 네트워크 ID와 호스트 ID로 이루어져 있다.
  - IP 주소를 효율적으로 관리하기 위해 클래스라는 개념이 도입되었다.
  - 네트워크 ID가 처음부터 몇개의 영역(옥텟)을 차지하는지에 따라 A, B, C, D, E 클래스로 나뉜다.
  - 네트워크와 호스트 ID를 구분하기 위해 서브넷 마스크를 사용한다. (표기는 동일)
  - 이 덕분에 호스트 IP 개수와 네트워크 개수를 유동적으로 할당할 수 있다.

#### IP 프로토콜의 한계점
- 비신뢰성
  - 전송한 패킷의 손상여부와 패킷 전달 순서가 보장되지 않는다.
- 비연결성
  - 송신자와 수신자가 연결되지 않기 때문에 송신자가 서비스 불능 상태여도 패킷을 전달한다.

## PORT
  - 같은 IP 내에서 패킷을 전달받을 프로세스를 구분하는 데 사용하는 번호를 의미한다.
  - PORT 번호는 16비트로 이루어져 있어 총 65536개의 포트 번호가 존재한다.

#### 잘 알려진 포트
- HTTP : 80
- HTTPS : 443
- FTP : 20
- SSH : 22
- ...

## CIDR
  - Classless Inter-Domain Routing (클래스 없는 도메인간 라우팅 기법)
  - CIDR(사이더)는 기존의 IP 주소 할당 방식이었던 클래스보다 더 유연하게 네트워크와 호스트 영역을 나눌 수 있다.
  - A.B.C.D/prefix 형태로 나타낸다.
    > 예시) 
  <br>IP 주소 : 191.166.179.210/25 (25비트까지가 네트워크 영역)
  <br>서브넷 마스크 : 255.255.255.128
  <br>네트워크 주소 : 191.166.179.210 & 255.255.255.128 = 191.166.179.128
  <br> 따라서 호스트 주소 범위는 191.166.179.128 ~ 191.166.179.255
    
## TCP와 UDP 차이
- TCP
  - 연결지향 프로토콜
  - 클라이언트와 서버가 3 way handshake를 통해 연결되었음을 보증한다.
  - 데이터의 전달을 보장한다.
  - 데이터의 전송 순서를 보장한다.
  - 데이터가 손실되면 재전송을 요청한다.
  - 서버와 클라이언트가 1대1로 연결된다.
  - 따라서 신뢰성이 높지만 3 way handshake와 같은 작업으로 인해 속도가 느리다.


- UDP
  - 비 연결지향 프로토콜
  - 클라이언트와 서버가 연결되지 않는다.
  - 데이터의 전달을 보장하지 않는다.
  - 데이터의 전송 순서를 보장하지 않는다.
  - 데이터가 손실되어도 재전송을 요청하지 않는다.
  - 서버와 클라이언트가 1대1, 1대N, N대N 으로 연결된다.
  - 따라서 신뢰성이 낮지만 빠른 속도를 보장해 실시간 서비스에 자주 사용된다.

## Web Server와 WAS의 차이
- 웹 서버(Web Server)
  - 컴퓨터에 저장되어 있는 정적 리소스(IMAGE, HTML, CSS, JS) 파일을 제공한다.
  - 항상 동일한 페이지를 반환한다.


- 웹 애플리케이션 서버(WAS)
  - 들어온 요청에 따라 서버내의 로직을 거쳐 동적인 컨텐츠를 제공한다.
  - 정적 리소스를 제공하는 웹 서버의 기능을 포함한다.


- WAS가 웹 서버의 기능을 포함하고 있다면 WAS 하나만 두면 될텐데 웹 서버도 함께 사용하는 이유는?
  - 효율적으로 리소스를 관리하기 위함이다. 
  - WAS는 DB 접근, 비즈니스 로직 등의 작업을 수행하기 때문에 정적 컨텐츠 요청은 웹 서버에서 처리하도록 함으로써 서버의 부하를 방지한다.

# OSI 7계층과 TCP/IP 4계층 모델

> 컴퓨터 간 네트워크 통신 과정에서 모든 컴퓨터는 기본적으로
OSI 7계층 모델 혹은 TCP/IP 4계층 모델을 이용한다.

### OSI 7계층
국제표준화기구에서 개발한 네트워크 통신 표준 모델로써,
네트워크 통신의 과정을 7단계의 계층으로 나눈 설계를 의미한다.

- 계층별로 해야할 일이 명확해진다.
- 각 층별로 필요한 데이터를 표준화할 수 있다.

### OSI 7계층 모델의 계층 별 역할

- 계층 7: 응용 계층(Application Layer)
  - 웹 서비스의 UI부분, 사용자의 입출력(I/O)을 담당한다.
  - 프론트, 백엔드 서버가 바로 이 레이어 위에서 동작한다.


- 계층 6: 표현 계층(Presentation Layer)
  - 이미지 압축, 데이터 암호화 등 계층 간 데이터를 적절히 표현하는 부분을 담당한다.


- 계층 5. 세션 계층(Session Layer)
  -  통신 세션을 구성한다.


- 계층 4. 전송 계층(Transport Layer)
  - 컴퓨터로 들어온 네트워크 데이터를 어느 포트(어느 프로세스)로 보낼지 담당한다.


- 계층 3. 네트워크 계층(Network Layer)
  - IP 주소를 사용하여 네트워크 데이터를 어느 컴퓨터로 보낼지 담당한다.
  - 라우터라는 기계가 관여한다.


- 계층 2. 데이터 링크 계층(Data Link Layer)
  - 네트워크 카드의 MAC 주소를 사용해 네트워크 데이터를 어느 컴퓨터로 보낼지 담당한다.


- 계층 1. 물리 계층 (Physical)
  - 디지털 데이터를 네트워크 전선이 이해할 수 있는 아날로그 신호로 변환해 흘려보낸다.
  - 아날로그를 디지털 신호로 변환하는 역할도 한다.

### TCP/IP 4계층 모델
OSI 7계층 모델은 네트워크 통신을 표준화하기 위해 제안된 개념적인 모델이다.
실제로 인터넷 통신은 TCP와 IP에 기반한 TCP/IP 통신을 사용한다.
그리고 이 통신에 특화된 네트워크 통신 계층 모델을 TCP/IP 4계층 모델이라고 한다.
OSI와 유사하지만 4단계로 간소화 되어있다.

- 계층 4 : 응용 계층(Application Layer)
  - OSI 7계층 모델의 7,6,5(응용, 표현, 세션) 계층 기능과 같다.
  - HTTP, SSH 같은 프로토콜이 여기에서 사용된다.

- 계층 3 : 전송 계층(Transport Layer)
  - OSI 7 계층 모델의 4(전송) 계층 기능과 같다.
  - TCP, UDP와 같은 프로토콜이 여기에서 사용된다.

- 계층 2 : 인터넷 계층 (Internet Layer)
  - OSI 7계층 모델의 3(네트워크) 계층 기능과 같다.
  - 컴퓨터 간 라우팅을 담당한다.

- 계층 1: 네트워크 인터페이스 계층 (Network Interface Layer)
  - OSI 7계층 모델의 2, 1(데이터 링크, 물리)계층 기능과 같다.
  - 네트워크 통신의 물리적인 부분들을 주로 포함한다.

### HTTP
Hyper Text Transfer Protocol,
텍스트 문서를 주고 받기 위해 만들어진 프로토콜이다.

### HTTPS
HTTP에 SSL(보안을 위한 레이어)이 추가된 것이다.
보안에 취약한 HTTP를 보완하기 위해 응용 계층과 전송 계층 사이에 보안 레이어가 추가되었다.
